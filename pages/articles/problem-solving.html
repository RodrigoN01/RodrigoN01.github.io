Problem Solving

Blocked on a simple problem

In some of the 'kata' exercises, I've gotten stuck on a few simple problems, sometimes because I didn't remember the methods to use and sometimes because I knew the syntax but I didn't know how to put them together to solve the problem.

In this particular case, I wasn't necessarily blocked, I knew the solution, but I wanted to solve it in a different and more elegant way.

In order to solve this problem, first, I've 'googled' it and I've got pretty close to finding the solution, there was just a few syntax errors I was missing,then afterwards I had to ask the coaches for help.

Gradebook Problem

Initially I was hard coding the objects.

 let gradebook = {
   Elizabeth : {
     testScores: scores[3]
   },
   Wiremu: {
     testScores: scores[2]
   },
   Susan: {
     testScores: scores[1]
   },
   Joseph: {
     testScores: scores[0]
   }
 }

However, I knew there was a way to create the objects by looping through the 'students' array. And this was my elegant solution for this part. 

 let gradebook = {}

 for (let i = 0; i < students.length; i++){
   gradebook[students[i]] = {testScores: scores[i]}
 }

It outputs the same result but it is more scalable, in case I want to add another value to the object and it is more elegant using logic.

I've learnt that there are many different ways to solve the same problem, many different approaches to tackle the same problem. It can be frustrating at times, and most of the times you will end up refactoring your solution to a cleaner and more elegant code.

Pseudocode

Your are basically writing a step-by-step 'recipe' of your code in your own language before actually writing code.

This method is my second favorite, behind 'Googling', it can be used in conjunction with other methods and it is very handy in more complex cases where you have to break down your solution in small parts.

The Rubber Ducky Method

In the software development world, the Rubber Ducky Method is known to be very helpful. The developer would carry a rubber duck around and when he is debugging a code, he would pull the little ducky and start explaining the problem to it. In many cases, you would find a solution just by verbalizing your thought process.

This method was a surprise for me, It was the first time I've heard of it. I will try to use it some day. I may have done this unconsciously.

Console.logging

By using the 'console.log' method, you get useful information to understand the code you just wrote and if your code is running correctly.

I use it a lot, it is almost as if I was testing each part of the code to see if I'm going to the right direction.

Googling

My favorite method!

I tend to say, that when I don't know something, "I am one google way from knowing it". It is the 'oracle' of the world, every thing is there not only in the programing world, but in many different areas. Specifically in the programming world, if encounter a problem, there is a great chance that someone else has already encountered that same problem and there will be a solution posted on 'Stack Overflow' or on another forum/community on Google.

Since it is owned by Google, I will include YouTube as part of the same method, in many cases, I search on Youtube first before asking Google. I appreciate the visual understanding of what I'm after.

So yes, trust the oracle and you will get what you were searching for.

Asking peers for help

I've used this in the past when I was studying by myself doing online courses. I've joined several programming communities on Discord, Slack, Youtube etc. It can be very helpful, but when it gets too complex, unless you are in person, the back and forth on the 'chat' can be frustrating. So I try not to use this method too often, unless I'm in person.

Asking coaches for help

Similarly to the circumstances above, unless I can talk to the person in front of me, I don't like asking for help via 'chat', exception when it is a simple question. A more complex problem can become frustrating for both sides.

Improving with reflection

I've been practicing this method since I have started the course at EDA, and it has been showing very effective, you take your time to step back and review your efforts, your mistakes and your achievements. It can help you develop your skills and gives you the opportunity to compere your today-self with your yesterday-self and therefore evolve.
